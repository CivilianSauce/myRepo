To use **7-Zip** (via `7z`) on Linux, you typically install the **p7zip** package and then run commands in the terminal. Here’s a clear guide:

---

## 🔧 1. Install 7z (p7zip)

On different Linux distros:

* **Ubuntu / Debian**

  ```bash
  sudo apt update
  sudo apt install p7zip-full
  ```

* **Fedora**

  ```bash
  sudo dnf install p7zip p7zip-plugins
  ```

* **Arch Linux**

  ```bash
  sudo pacman -S p7zip
  ```

---

## 📦 2. Basic Usage

### ➤ Create an archive

```bash
7z a archive.7z file1 file2 folder/
```

* `a` = add files to archive

---

### ➤ Extract an archive

```bash
7z x archive.7z
```

* `x` = extract with full paths

OR:

```bash
7z e archive.7z
```

* `e` = extract without folder structure

---

### ➤ List contents

```bash
7z l archive.7z
```

---

### ➤ Test archive integrity

```bash
7z t archive.7z
```

---

## 🔐 3. Password Protection

### Create encrypted archive:

```bash
7z a -p archive.7z folder/
```

* It will prompt for a password

OR specify directly:

```bash
7z a -pMyPassword archive.7z folder/
```

---

## ⚙️ 4. Compression Levels

```bash
7z a -mx=9 archive.7z folder/
```

* `-mx=0` → no compression
* `-mx=9` → ultra compression (slow but small)

---

## 📁 5. Extract to Specific Directory

```bash
7z x archive.7z -o/output/path
```

---

## 🧠 Tips

* `.7z` gives better compression than `.zip`
* Works with many formats: `.zip`, `.tar`, `.gz`, `.rar` (extract only)
* Use `man 7z` to see full documentation

# Where?

Perbedaannya dengan ZIP dan lainnya bukan sekadar “nama format”, tapi **algoritma kompresi + tujuan desainnya**. Beberapa fokus ke kecepatan, beberapa ke rasio kompresi, beberapa ke fleksibilitas.

Aku ringkas tapi tetap jelas:

---

## 1. ZIP

* Algoritma utama: **DEFLATE**
* Karakter:

  * Cepat
  * Kompatibilitas sangat luas (Windows, Linux, dll.)
  * Kompresi “cukup”, tapi bukan terbaik

**Cocok untuk:**

* Kebutuhan umum
* Sharing file
* Situasi yang butuh universal support

---

## 2. RAR

* Algoritma: proprietary (milik Eugene Roshal)
* Karakter:

  * Kompresi lebih baik dari ZIP
  * Ada fitur recovery (bisa memperbaiki file rusak)
  * Mendukung split archive (part1, part2, dst)

**Kekurangan:**

* Tidak open-source (alat buat bikin RAR terbatas)

**Cocok untuk:**

* Arsip penting
* Distribusi file besar

---

## 3. 7z

* Algoritma utama: **LZMA / LZMA2**
* Karakter:

  * Kompresi **sangat tinggi** (sering terbaik)
  * Tapi lebih lambat
  * File bisa jauh lebih kecil dibanding ZIP

**Cocok untuk:**

* Menghemat storage semaksimal mungkin
* Backup

---

## 4. TAR

* Ini sering disalahpahami

**TAR bukan kompresi**, tapi:

* Hanya “menggabungkan file” (archive)
* Tidak mengecilkan ukuran

Biasanya dipakai bersama:

* tar + gzip → `.tar.gz`
* tar + xz → `.tar.xz`

---

## 5. GZIP

* Algoritma: DEFLATE (mirip ZIP)
* Karakter:

  * Cepat
  * Efisien untuk teks
  * Hanya untuk **satu file** (makanya sering digabung TAR dulu)

---

## 6. XZ

* Algoritma: LZMA2 (mirip 7z)
* Karakter:

  * Kompresi sangat tinggi
  * Lebih lambat
  * Umum di Linux (misalnya package distro)

---

## Perbandingan sederhana

| Format | Kompresi          | Kecepatan    | Fitur           | Catatan           |
| ------ | ----------------- | ------------ | --------------- | ----------------- |
| ZIP    | Sedang            | Cepat        | Basic           | Paling kompatibel |
| RAR    | Tinggi            | Sedang       | Recovery, split | Proprietary       |
| 7z     | Sangat tinggi     | Lambat       | Banyak opsi     | Paling efisien    |
| TAR    | ❌ (tidak kompres) | Sangat cepat | Gabung file     | Perlu gzip/xz     |
| GZIP   | Sedang            | Cepat        | Simple          | 1 file            |
| XZ     | Sangat tinggi     | Lambat       | Efisien         | Linux-oriented    |

---

## Kenapa hasilnya bisa beda jauh?

Karena tiap algoritma punya pendekatan berbeda:

* **DEFLATE (ZIP/GZIP)**
  → cepat, tapi tidak terlalu “dalam” mencari pola

* **LZMA (7z/XZ)**
  → mencari pola lebih kompleks dan jauh
  → hasil lebih kecil, tapi butuh waktu & RAM lebih

---

## Intinya

* ZIP = praktis & universal
* RAR = fitur + kompresi lebih baik
* 7z/XZ = maksimalin ukuran sekecil mungkin
* TAR = cuma bungkus, bukan kompres

---

Kalau kamu kaitkan ke pengalamanmu tadi:

* VDI 4GB → 1GB
  → kalau pakai **7z atau xz**, kemungkinan bisa lebih kecil lagi dibanding ZIP

Kalau kamu mau, aku bisa bantu rekomendasikan “format terbaik” tergantung kebutuhanmu (backup, koleksi, atau distribusi).
